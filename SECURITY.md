# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 0.1.x | Yes |

## Reporting a Vulnerability

Use GitHub private vulnerability reporting when available. If it is not available, open a minimal public issue that does not include PHI, credentials, endpoint URLs, or institutional details.

## Data Handling Expectations

- Run real radiology report data only inside authorized environments.
- Use synthetic reports for demos and automated tests.
- Review extracted findings before sharing them externally.
- Flagged items requiring manual review should be verified by a qualified clinician.
